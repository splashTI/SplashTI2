Installation


Download and unzip the application into TM1 Server folder. Copy ssl folder content from TM1 installaton directory (C:/Program Files/ibm/cognos/tm1_64/bin64/ssl) to ../splashTI/ssl. Copy TM1JavaApi.jar into ../splashTI/lib folder from TM1 installation directory (C:/Program Files/ibm/cognos/tm1_64/bin/classes/TM1JavaApi.jar). Be aware: to be able execute .exe files it's required to set up Java Environment Variables.

Configuration


Change log folder if it's required. Specify configuration for password encryption: password and vector. For both (for password and for vector) it must be 16digit string. Numbers and alphabet characters are allowed only. Recommended logging level is CONFIG. To be able see log in cmd line console set CONSOLEHANDLER as a handler.

SplashTI2


Firstly, it's required to construct a batch txt file in a main TI process. SplashTI2 process line syntax is similar to TM1RunTI syntax. -pwdencrypted "Y" is a SplashTI2 parameter, which allows to use encrypted password. Don't be confused by specifying TM1RunTI.exe as a first process line parameter, it would be ignored anyway. It's done in order to make transition from TM1RunTI to SpashTI2 easier, and basicaly you can specify any string instead of "C:\Program Files\ibm\cognos\tm1_64\bin64\tm1runti.exe".

Secondly, run SplashT2 from command line. Threads Number is an optional parameter. Accepted Integer numbers. If Threads Number is negative or zero, SplashTI2 will use availiable logical processors + Thread Numbers. So if Threads Number = -1 and there are 12 logical processors on server, Splash TI will run a batch file in 11 threads concurently. Run ID is an optional parameter, which can be used after as argument for SplashTImanager.

SplashTImanager


SplashTImanager is an application for stopping SplasTI2 procesess execution. Run ID should be unique, otherwise clean tmp folder. Be Aware: to be able use .exe file it's required to set up Environment variables for Java.
